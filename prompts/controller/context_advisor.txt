---

输出 JSON：
- memory_strategy: "minimal"|"normal"|"deep_recall"
- session_mode: "comment_focus"|"video_focus"|"none"
- session_anchor: 可选，如"继续 老观众 上次聊的话题"
- extra_instructions: 可选，最多 2 条短句指令
- corpus_style: 可选，必须从可用语料风格标签中选择；不需要时填空字符串
- corpus_scene: 可选，必须从可用语料场景标签中选择；不需要时填空字符串
- topic_assignments: 可选，格式 {{"弹幕ID": "话题ID"/"new_话题名"/"none"}}
- 只输出 JSON 对象本体，不要解释，不要代码块，不要把 JSON 再包成字符串

规则：
- 观众有"未了话头"或"回钩线索" → memory_strategy="deep_recall"
- 弹幕像在打关系牌（还记得/老粉/好久不见） → 填 session_anchor 锁定该观众
- 会员发言 → 倾向 memory_strategy="deep_recall"
- 普通闲聊 → memory_strategy="normal"，不需要 anchor
- 有弹幕时 → session_mode="comment_focus"
- 竞品话题 → extra_instructions 提醒先亮态度
- 知识命中、存在性问题、深回忆关系牌默认不要触发语料，corpus_style/corpus_scene 留空
- 短弹幕吐槽、接梗、轻互动、冷场打破沉默时，可以考虑触发语料
- 如果触发语料，style 和 scene 至少填一个；拿不准宁可留空
- 只能从给定标签里选，不能自造标签

示例：
{{"memory_strategy":"normal","session_mode":"comment_focus"}}
{{"memory_strategy":"deep_recall","session_mode":"comment_focus","session_anchor":"继续 老观众 上次聊的话头","extra_instructions":["先接住关系牌再顺着聊"]}}
{{"memory_strategy":"normal","session_mode":"comment_focus","corpus_style":"搞笑","corpus_scene":"互动"}}
